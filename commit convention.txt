① 커밋 메시지 구조
  <type> : 커밋 제목
  <커밋 설명>
  <이슈 트래커(옵션)>

② type
  feat : 새로운 기능 추가
  refactor : 리팩토링(결과는 같지만 코드를 개선함)
  fix : 버그 수정
  style : 스타일 수정
  comment : 주석 수정 및 삭제
  Docs : 문서와 관련된 모든 것
  Implement : 코드가 추가된 정도보다 더 주목할만한 구현체를 완성시켰을 때
  Remove : 파일을 삭제하는 작업만 수행한 경우

③ 이슈트래커
  : 해당 커밋이 어떤 이슈를 해결하였는지 표기. (혹은 해결하지 못한 이슈도 작성 가능)
